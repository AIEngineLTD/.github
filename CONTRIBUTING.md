# Contributing to AIEngine Projects

Thank you for your interest in contributing to AIEngine projects! We welcome contributions that align with our principles of innovation, quality, and impact.

## Our Principles

- **Innovation** – Push boundaries with emerging AI and product patterns.
- **Collaboration** – Work transparently with founders, operators, and domain experts.
- **Quality** – Sweat the details, from pixel-perfect design to type-safe code.
- **Excellence** – Deliver outcomes that exceed the brief.
- **Integrity** – Communicate honestly and build long-term trust.
- **Impact** – Measure success by the value created for users and businesses.

## How to Contribute

### Reporting Issues

Before creating an issue, please:
1. Check if the issue already exists
2. Use a clear, descriptive title
3. Provide detailed information about the problem
4. Include steps to reproduce (for bugs)
5. Specify your environment (OS, browser, versions, etc.)

### Proposing Features

When proposing a new feature:
1. Open an issue first to discuss the idea
2. Explain the problem it solves and the value it adds
3. Describe your proposed solution
4. Consider implementation complexity and maintainability

### Submitting Pull Requests

1. **Fork the repository** and create a branch from `main`
2. **Follow our coding standards**:
   - Use TypeScript for type safety
   - Write clear, self-documenting code
   - Add comments for complex logic
   - Ensure accessibility and performance
3. **Write tests** for new features or bug fixes
4. **Update documentation** as needed
5. **Keep commits focused** and messages clear
6. **Test your changes** thoroughly before submitting

### Code Style

We value:
- **Type safety** – Prefer TypeScript with strict mode
- **Accessibility** – Follow WCAG guidelines
- **Performance** – Optimize for speed and efficiency
- **Maintainability** – Write code that's easy to understand and modify

### Review Process

1. All PRs require review before merging
2. Address feedback promptly and professionally
3. Be open to suggestions and alternative approaches
4. We aim to review PRs within 2-3 business days
