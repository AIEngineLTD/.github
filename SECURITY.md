# Security Policy

## Supported Versions

We actively support security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability, please follow these steps:

1. **Do not** open a public issue
2. Email us at [contact@aiengineltd.com](mailto:contact@aiengineltd.com) with:
   - A clear description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact and severity
   - Any suggested fixes or mitigations

3. We will:
   - Acknowledge receipt within 48 hours
   - Provide an initial assessment within 5 business days
   - Keep you informed of our progress
   - Credit you (if desired) once the vulnerability is resolved

## Security Best Practices

When working with AIEngine projects:

- Keep dependencies up to date
- Follow secure coding practices
- Never commit secrets or API keys
- Use environment variables for sensitive configuration
- Validate and sanitize all user inputs
- Follow the principle of least privilege

## Our Commitment

We are committed to:
- Responding to security reports promptly
- Fixing vulnerabilities in a timely manner
- Maintaining transparency about security issues
- Following responsible disclosure practices

Thank you for helping keep AIEngine projects secure!
